Cheat Sheet de SQL

Consultas de Agregación
1 / 11

¿Qué es una Agregación?

Es una operación que toma un conjunto de filas (muchos valores) y devuelve un único valor de resumen.

Son la base para generar reportes, estadísticas y análisis de negocio.

2 / 11

COUNT: Contar Filas

Cuenta el número de filas que cumplen una condición. `COUNT(*)` cuenta todas las filas de la tabla.

-- ¿Cuántos productos hay en total en el inventario?
SELECT COUNT(*) FROM Productos;

-- ¿Cuántos empleados hay en el departamento de 'Ventas'?
SELECT COUNT(*) FROM Empleados WHERE Departamento = 'Ventas';
3 / 11

SUM: Sumar Valores

Suma todos los valores de una columna numérica.

-- ¿Cuál es el valor total de todas las ventas?
SELECT SUM(Monto) FROM Ventas;

-- ¿Cuál es el total de salarios que paga la empresa?
SELECT SUM(Salario) FROM Empleados;
4 / 11

AVG: Calcular el Promedio

Calcula el valor medio (promedio) de una columna numérica.

-- ¿Cuál es el precio promedio de nuestros productos?
SELECT AVG(Precio) FROM Productos;

-- ¿Cuál es la edad promedio de los empleados?
SELECT AVG(Edad) FROM Empleados;
5 / 11

MIN y MAX: Valor Mínimo y Máximo

Devuelven el valor más bajo (MIN) o más alto (MAX) de una columna.

-- ¿Cuál es el producto más barato y el más caro?
SELECT MIN(Precio), MAX(Precio) FROM Productos;

-- ¿Cuál fue la fecha de la primera y la última venta?
SELECT MIN(FechaVenta), MAX(FechaVenta) FROM Ventas;
6 / 11

GROUP BY: Agrupar para Analizar

La verdadera potencia aparece al combinar agregaciones con `GROUP BY`. Esta cláusula agrupa filas con el mismo valor para que puedas aplicar funciones a cada grupo.

-- ¿Cuántos productos hay POR CADA categoría?
SELECT Categoria, COUNT(*)
FROM Productos
GROUP BY Categoria;
7 / 11

GROUP BY con Múltiples Funciones

Puedes aplicar varias funciones de agregación al mismo tiempo sobre los grupos.

-- Para cada categoría, dame el total de productos y su precio promedio.
SELECT
    Categoria,
    COUNT(*) AS TotalProductos,
    AVG(Precio) AS PrecioPromedio
FROM Productos
GROUP BY Categoria;
8 / 11

HAVING: Filtrar Grupos

`HAVING` es como un `WHERE`, pero se aplica **después** de agrupar los datos. Sirve para filtrar los grupos que no cumplen una condición.

-- Muéstrame solo las categorías que tienen MÁS de 10 productos.
SELECT Categoria, COUNT(*)
FROM Productos
GROUP BY Categoria
HAVING COUNT(*) > 10;
9 / 11

WHERE vs. HAVING

`WHERE` filtra filas antes de la agregación.
`HAVING` filtra grupos después de la agregación.

-- Ventas totales por país, pero solo de clientes de 'Europa'
-- y solo si el total de ventas supera 10000.
SELECT Pais, SUM(Monto)
FROM Ventas
WHERE Continente = 'Europa' -- Filtra filas primero
GROUP BY Pais
HAVING SUM(Monto) > 10000; -- Filtra grupos después
10 / 11

Resumen Final

Con las funciones de agregación, `GROUP BY` y `HAVING`, puedes transformar largas listas de datos en resúmenes claros y útiles.

Has pasado de "ver datos" a "entender datos".
